StrmCompanion — Help

A quick guide to all three features
StrmCompanion is an Emby plugin that adds three features Emby does not provide out of the box. Each feature lives on its own tab at the top of this page.
skip_next Intro Detection

Listens to the audio of TV episodes across a season and finds the part that repeats in every episode — the opening theme. Once detected, a Skip Intro button will appear in the Emby player so viewers can jump past it with one click.

info Media Info

When your library contains .strm files — small text files that point to an online stream (e.g. IPTV) — Emby cannot read their technical details on its own. This feature connects to each stream and retrieves the codec, resolution, audio format and other properties so they show up correctly in Emby.

merge Merge Version

If you have the same movie or episode in multiple qualities (for example both a 1080p and a 4K copy), Emby can group them as alternate versions of the same title. This feature does that automatically by matching titles that share the same IMDb or TVDb ID.

Intro Detection works by creating a short audio fingerprint for each episode and then comparing them. The segment of audio that is identical across multiple episodes is identified as the intro. No internet connection is required — all analysis runs locally on your server.
build Requirements

FFmpeg with the Chromaprint audio muxer must be available on your server. You can use the FFmpeg bundled with Emby or point to a custom installation in Settings. At least two episodes in the same season are needed to detect a match.

settings Configure settings (first time)
  1. Click Intro Detection — Settings to expand the settings panel.
  2. Fingerprint cache path — leave empty to use the default Emby plugin folder, or enter a custom path if you want the fingerprint files stored elsewhere.
  3. FFmpeg path — leave empty to use Emby's built-in FFmpeg. Only change this if you need a specific version.
  4. Adjust Fingerprint duration (how many minutes of each episode to analyse — 5 minutes is enough for most shows).
  5. Set Minimum intro length to roughly 60–80 % of your show's actual intro length.
  6. Click Save settings.
play_arrow Run Intro Detection
  1. Under Select what to analyse, choose a Series from the dropdown.
  2. Choose a Season, or leave it as All seasons to queue the entire series.
  3. Click Add to queue. Repeat for more series if you like.
  4. Click Run to start the analysis. A progress bar and live status will appear.
  5. When done, the results table shows each episode and whether an intro was found.
playlist_play View and manage markers

After a successful run, expand Current intro markers at the bottom of the page to see a list of all detected markers. You can delete markers for a single season or an entire series using the red buttons. Clear fingerprint cache removes the stored audio fingerprint files from disk but leaves the markers intact.

Tip: If the intro is not detected correctly, try lowering the Hamming threshold (stricter matching) or increasing the Fingerprint duration so more audio is analysed. For low-quality video sources, a higher threshold (e.g. 12–16) may help.
A .strm file is a plain text file that contains a single URL pointing to an online stream — commonly used for IPTV channels or remote video files. Emby adds these items to your library but cannot probe the stream for technical details at scan time. Media Info fills that gap by connecting to each stream and storing the result back in Emby.
settings Configure settings (first time)
  1. Click Media Info — Settings to expand the settings panel.
  2. Under Libraries to scan, tick every library that contains .strm files.
  3. Set Concurrent scans — this controls how many streams are probed at the same time. Use 1–3 to avoid being rate-limited by your IPTV provider.
  4. Optionally enable Auto-scan new .strm files so that newly added items are probed automatically without manual action.
  5. Click Save settings.
search Run a scan
  1. The stats bar at the top shows how many movies and episodes have already been scanned and how many are still missing info.
  2. Click Start Scan to begin. Only items with missing info are probed — already-scanned items are skipped.
  3. A progress bar shows how the scan is going. Click Cancel at any time to stop.
  4. When done, the results show each item and whether media info was successfully retrieved or not.
  5. Click Refresh Stats at any time to update the counts.
Tip: A result of No info usually means the stream was unreachable at that moment (e.g. the IPTV server was down or the channel is geo-blocked). Simply run the scan again later to retry those items.
Emby supports grouping multiple files of the same title as alternate versions — for example a 1080p copy and a 4K copy of the same movie. Normally you have to merge these manually one by one. Merge Version does it automatically across your entire library by matching items that share the same IMDb ID (movies) or TVDb ID (series).
settings Configure settings (first time)
  1. Click Merge Version — Settings to expand the settings panel.
  2. Set Movies scope:
    • Global — looks for duplicates across all your movie libraries combined.
    • Per-library — each library is checked independently (duplicates must be in the same library to be merged).
  3. Set Series scope:
    • Disabled — only movies are processed.
    • Enabled — also merges duplicate episodes and refreshes series that share a TVDb ID in TV libraries with auto-grouping turned on.
  4. Optionally enable Auto-detect new movies and episodes so a merge runs automatically 30 seconds after a new item is added.
  5. Click Save settings.
merge Run a merge
  1. Click Run Merge. The plugin will scan your libraries for items with matching IDs.
  2. A progress bar and a live table show each item as it is processed.
  3. When done, the results table lists every item and whether it was merged, already grouped, or failed.
  4. The stats bar above shows a summary: how many groups were Found, Merged, and how many Failed.
Note: For a merge to succeed, both items must already have an IMDb or TVDb ID assigned in Emby. If an item shows as Failed, open that item in Emby, trigger a metadata refresh, and then run the merge again.
